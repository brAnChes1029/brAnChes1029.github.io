2026

day1

Ferris Wheel

full

+ 是往比自己大的連，先把全部是 + 的零件分離出來，編號由小往大連。
- 是往比自己小的連，把剩下的 - 的零件編號由大往小連。

這時候會有兩條鍊，把 + 編號最大的連到 - 編號最大的，- 編號最小的連到 + 編號最小的。
如果這裡連不上去 ( 也就是 $0$ 號是 - 或者 $N-1$ 號是 + 的情況 ) 就輸出 NO。

Ovenmasters

full

首先每堆披薩第一個先加入答案，一個想法是可以從所有第二個的披薩和生氣的披薩這個集合裡面每次挑 index 最小的，然後看看能不能成功加進答案，之後再更新集合。
不過這樣是 $\mathcal{O}(N^2 \log N)$。

這邊有幾個觀察 :

  1. 先把所有桌子按照擺在最上面的披薩編號 $a[i]$ 由小到大排好，編號 $x$ 的披薩目標是放在第 $i$ 桌，必須先滿足 $a[i]<x<a[i+1]$ 這個條件，並且把 $x$ 號成功放上去之後桌上披薩的大小順序不會改變。
  2. 假設編號 $x$ 的披薩目標是放在 $i$ 桌，編號 $y$ 的披薩目標是放在 $j$ 桌 ( $i<j$ )，且這兩個披薩都有滿足上面的條件，一定有 $x<y$，而且一定是先放 $x$。
  3. 生氣的披薩在答案裡的順序一定是由小到大，因為如果在某一時間 $i$ 號披薩生氣地離開，所有編號比 $i$ 小的披薩也會生氣地離開。

接下來就可以用 priority_queue 存現在滿足條件的的披薩們各自目標是放到哪一桌上 ( 由桌子編號小到編號大 )，把披薩擺上去，然後更新滿足條件的披薩放進 priority_queue ( 如果剛剛在 $i$ 桌放了一個新披薩，$i$ 和 $i-1$ 號桌都要更新，因為 $a[i]$ 變大了，可能會使 $i-1$ 號桌的候選披薩符合條件 )。
至於生氣離開的披薩就看現在這個桌子的 $min=a[0]$ 是多少，就可以再用一個指針把所有 $x<a[0]$ 的生氣披薩放進答案。

Biscuits

subtask 1

答案是 $\lfloor \frac{N}{2} \rfloor$。

subtask 3

這裡保證 :

$$W_0 \geq W_1 \geq \dots \geq W_{N-1}$$

Bianca 可以保證她的得分至少為：

$$W_1 + W_3 + W_5 + \dots + W_{N-1-(N \bmod 2)}$$

以下用數學歸納法證明 ( 完蛋我不會數學 )。

對於 $N = 1$，Bianca 可以保證她的分數非負。

假設對於大小為 $1 \dots N-1$ 的餅乾堆皆成立。當在一個大小為 $N$、權重為 $W_0, W_1, \dots, W_{N-1}$ 的餅乾堆上玩遊戲時，Bianca 會：

  • Aurora 選擇 $X = 1$，那麼 Bianca 就選擇 $Y = 0$，並且吃掉總權重至少為 $W_0 + (W_2 + W_4 + \dots) \geq W_1 + W_3 + W_5 + \dots$ 的餅乾。
  • 不然 Bianca 就選擇 $Y = 1$，並且吃掉總權重至少為 $W_1 + (W_3 + W_5 + \dots)$ 的餅乾。

Aurora 可以總是選擇 $X = 0$ 確保 Bianca 吃掉的餅乾總權重最多為 $W_1 + W_3 + W_5 + \dots$。

subtask 2 & 4

考慮 DP。
令 $dp[i]$ 代表考慮 $i, i+1, … , N-1$ 塊餅乾，Bianca 可以獲得的最高權重。
可以對每個 $dp[i]$ 枚舉 $X$、$Y$ 來轉移，每次 query 都重新 DP 一次。

subtask 5 & 6

優化剛剛的 DP。
Aurora 會選一個 $X$，Bianca 必須選一個不一樣的 $Y$，然後從 $Y$ 轉移，所以 Aurora 可以把最好的轉移點擋掉，因此可以直接用 second max 轉移，這樣 DP 就變 $\mathcal{O}(N)$ 了。

full

令 $B[i]=\text{secondmax}_{i \le j}(W[j]+B[j+1])$。
令 $A[i]=\text{max}_{i \le j}(W[j]+B[j+1])$。
令 $D[i]=A[i]-B[i]$，$S[i]=A[i]+B[i]$。

現在要算 $S[i]$，因為 $S[i+1]=A[i+1]+B[i+1]$，而且現在多了新的轉移點 $W[i]+B[i+1]$，他一定比 $B[i+1]$ 大，如果他也比 $A[i+1]$ 大的話就會 $A[i]=W[i]+B[i+1]$，否則就會 $B[i]=W[i]+B[i+1]$，反正總有 $S[i]=A[i+1]+B[i+1]+W[i]=\sum_{j=i}^{N-1} W[j]$，所以 $A[0]+B[0]$ 會是全部餅乾的重量！

現在改算 $D[i]$，看上面的討論可以發現 $D[i]=|{D[i+1]-W[i]}|$，而且 $D[i]$ 的值域頂多到 $W_{max}$。

綜合上述，答案 $B[0]=\frac{\text{全部餅乾}-D[0]}{2}$。

subtask 3

使用像是線段樹那樣的想法。

第一輪，大家都在自己的 ID 上 write 1，然後按照 ID 0-1 2-3 4-5 6-7 這樣分組去看對方寫什麼，對方寫 $1$ 就可以知道這組共有 $2$ 人，看到 $0$ 代表這組只有一個人。
第二輪，01 23 45 67 各組決定出一個代表在 ID=0, 2, 4, 6 上 write 剛剛的總人數，然後按照 ID 01-23 45-67 這樣分組去看對方寫什麼，把對方寫的人數加進已知總人數。
第三輪，0123 4567 各組決定出一個代表在 ID=0, 4 上 write 剛剛的總人數，然後按照 ID 0123-4567 這樣分組去看對方寫什麼，把對方寫的人數加進已知總人數。
第四輪，最後的兩個人知道總人數，選一個代表把總人數 write 在 ID=0，之後其他人去 read。

各組要決定出一個代表才不會有多個人在同一個 ID 上 write，至於怎麼決定出一個代表，如果兩小組中都有人，就選編號小的。

這個想法可以推廣到 subtask 4，只能寫 0/1 的話，就把總人數用二進位多分幾次傳，可以拿到一些分數。

優化 1

上面說為了避免多人在同一地點 write，選擇讓編號小的作為代表負責 write，但這樣要花 $7$ 輪才可以傳一個 $N=100$ 這樣大的數字。

其實可以讓多人負責不同 bit，在同一輪 write。

要怎麼選出多人，可以維護每個人在現在組內的 ranking，合併左右兩邊的時候就把左邊的人 ranking 加上右邊的人數，然後讓 ranking 前 $7$ 大的人負責 $7$ 個 bit。

優化 2

這樣 merge 是一個樹狀的結構，但是在葉子的地方如果沒幾個人，就不需要用 $7$ bits 傳人數了，可以用少一點的 bit。
但是如果剛好遇上某一區人很多，又不能這樣節省了。

這裡可以用 random shuffle 把全部人初始分布位置變平均一點，random 的 seed 要是固定的 ( $N$ 個人的程式是分開執行的，但他們打亂的基準要是一樣的，才能成功打亂 )。
至於樹的哪一層要傳幾個 bit 就可能需要調整一下。

( 我是有成功這樣弄到 $100$ 分 )

Cakes

full 1

$A[i]$ 代表 $i$ 號配料有多少個，$max$ 代表最多個的配料有多少個。

現在決定一個美味度 $d$，這個美味度最少會做出 $L=\lceil \frac{max}{d} \rceil$ 個蛋糕，最多也只能做出 $R=\sum_{i=0}^{N-1} \lfloor \frac{A[i]}{d} \rfloor$ 個蛋糕，至於在 $[L,R]$ 中的個數都是可以被做出來的。
接下來枚舉所有美味度 $d$，計算這個美味度最多可以做出多少個蛋糕 ( 也就是 $R$ )。可以枚舉 $\lfloor \frac{A[i]}{d} \rfloor=1, 2, 3, …$ 的蛋糕分別有幾個 ( 使用前綴和 )，就可以用調和級數和在 $\mathcal{O}(N \log_e N)$ 算完。

詢問想做 $x$ 塊蛋糕時，因為美味度越大可以做出來的最多的蛋糕數 $R$ 越少，所以選擇最小的美味度 $d=\lceil \frac{max}{x} \rceil$，接下來再檢查美味度 $d$ 可以做出最多的蛋糕數量是不是大於 $x$ 就好了。

full 2

官解。

也可以不用先算出每個美味度最多可以做出多少蛋糕，因為 $\lceil \frac{max}{x} \rceil$ 最多只有 $\sqrt{max}$ 種不一樣的值 ( 見這邊 )，所以可以算過的存起來，遇到新的就暴力算。

full

每個連通塊紀錄 $minL$、$minR$、$maxL$、$maxR$。
觀察到我要把區間 $[L,R]$ 和一個連通塊合併時，以下情況會無法合併 :

  1. $L<minL \text{ and } R<minR$
  2. $L>maxL \text{ and } R>maxR$

用一個 set 存連通塊，按照 $minL$ 由小到大排序，這裡有一個性質是 $minR$、$maxL$、$maxR$ 也都會分別遞增 ( 可以畫圖然後反證看看 )。
要加入區間 $[L,R]$ 時，先用二分搜找到第一個 $minL \leq L$，然後嘗試合併，一直重覆到無法合併，就直接停止 ( 根據上面說的性質，接下來的連通塊也都無法和 $[L,R]$ 合併 )。

另一個方向也要做一次一樣的事 ( 我是一直找第一個 $minR \leq R$ 然後合併看看 )。
